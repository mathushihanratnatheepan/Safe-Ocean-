Profile Page

Technical Description

The Profile page is built using semantic HTML5 elements with a focus on form design and user interaction. The page utilizes CSS Grid and Flexbox for a responsive layout that adapts to different screen sizes. A custom form is implemented with various input types including text, email, tel, select dropdowns, and textarea for comprehensive user data collection. The page features custom-styled form elements with focus states, hover effects, and validation styling. Internal CSS is used for page-specific styles while maintaining consistency with the global stylesheet.

Accessibility

The Profile page is designed with accessibility in mind. All form inputs have associated labels using the 'for' attribute for screen reader compatibility. The color contrast ratio meets WCAG 2.1 guidelines for readability. Interactive elements have clear focus states for keyboard navigation. The form includes proper input types (email, tel) for mobile device optimization and validation. Interest tags use semantic buttons that are keyboard accessible. The responsive design ensures usability across all device sizes without horizontal scrolling.

Feedback Page

Technical Description

The Feedback page implements a two-column layout using CSS Grid, featuring a contact information section and a comprehensive feedback form. The page uses semantic HTML elements including main, section, and form elements. Custom radio buttons are styled for the rating system using CSS pseudo-elements and the adjacent sibling selector. The FAQ section uses a grid layout for organized presentation of frequently asked questions. Flexbox is used for contact item layouts with icon integration using emoji characters for visual appeal without external dependencies.

Accessibility

The Feedback page incorporates multiple accessibility features. The form includes proper labels, placeholder text, and required field indicators. The rating system uses radio buttons with visible labels for clear selection feedback. Contact information is organized with clear headings and visual hierarchy. The FAQ section uses semantic headings (h3) for each question to aid screen reader navigation. All interactive elements have hover and focus states. The responsive design reorganizes content into a single column on smaller screens for improved mobile accessibility.

Content Page - Corals and Invertebrates

Technical Description

The Corals and Invertebrates content page follows a consistent structure with the other content pages in the project. It uses the hero section pattern with a full-viewport background image and overlay text. The page alternates between the 'marine' and 'Protect' section layouts to create visual variety while maintaining consistency. Each section uses a two-column grid layout with content and images. Internal anchor links (#invertebrates) enable within-page navigation. The newsletter section provides a call-to-action linking to the volunteer page.

Accessibility

This content page maintains high accessibility standards through semantic HTML structure including proper heading hierarchy (h1-h5). All images include descriptive alt text for screen reader users. The alternating layout pattern aids visual comprehension and reduces cognitive load. Internal links use clear, descriptive text. The "Go to top" button provides quick navigation for keyboard users. Text contrast and font sizes are optimized for readability across all devices. The responsive design ensures content remains accessible on mobile devices.
