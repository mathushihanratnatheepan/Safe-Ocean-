Splash Screen

Technical Description

HTML Techniques

The document begins with a proper HTML5 declaration and includes essential meta tags for charset, viewport, and an automatic redirection via a meta refresh (set to redirect after 4 seconds). Key content elements are grouped within a <div class="splash-screen"> that contains a prominent <h1> for the website title, a <p> for the mission statement, and additional <p> elements listing group members.

CSS Techniques

A full-screen background is achieved using the CSS background properties (e.g., background: url(...) no-repeat center center fixed; with background-size: cover), ensuring the image scales appropriately across devices. Flexbox is used to center the content both vertically and horizontally (display: flex; flex-direction: column; justify-content: center; align-items: center;). CSS animations via @keyframes (fadeIn, slideUp, and spin) create smooth transitions for text and a rotating loader, enhancing the user experience during the short splash duration.

Purpose & Justification

The combination of semantic HTML and dynamic CSS provides users with an engaging first impression, immediately communicates your site's mission, and smoothly transitions them to the Home page—all while maintaining a responsive design.

Accessibility

This splash screen uses semantic HTML elements like <h1> for the main heading and <p> for descriptive text, which improves readability for screen readers. The use of em for font sizes enhances accessibility by supporting text scaling. The viewport meta tag ensures responsiveness across devices, and the high contrast between the white text and darkened background improves visibility. However, accessibility can be further improved by adding role="banner" to the splash screen, using aria-label="Loading..." for the loader, and including skip links for better navigation. Additionally, providing a way to bypass the splash screen, like a skip button, would enhance user experience.

Volunteer Page

Technical Description

HTML Techniques

The header features a well-structured navigation bar using semantic tags such as <header>, <nav>, and an unordered list (<ul>) to list menu items. The main content is divided into clear sections: one for volunteering opportunities, another for user reviews, and a section for a feedback form—all wrapped in <section> elements to enhance readability and accessibility. Each volunteer opportunity is presented as a 'card' (using a <div class="card">), which includes images, icons (via Font Awesome), headings, descriptive text, and detailed information structured within sub-elements.

CSS Techniques

A CSS Grid layout is used to display volunteer cards in a responsive grid (grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));), ensuring cards adapt to various screen sizes. Hover effects and transitions (e.g., scaling the card with transform: scale(1.05); and adjusting box-shadow) add interactivity and visual feedback for clickable elements. Consistent typography, spacing, and a subtle color palette (with contrasting button colors) help maintain a clear visual hierarchy and an appealing, user-friendly interface.

Purpose & Justification

The structured layout and interactive design encourage users to explore volunteering opportunities, read reviews, and submit feedback. The use of semantic elements and responsive CSS supports both usability and the site's overarching mission of marine conservation.

Accessibility

Accessibility is addressed through the use of semantic HTML elements like <header>, <nav>, <section>, and <main>, which enhance readability for screen readers and improve content structure. The responsive grid layout ensures the website remains usable across various screen sizes, enhancing accessibility for users on different devices. Proper color contrast between text and backgrounds ensures readability for visually impaired users, while interactive elements like buttons and links are styled with clear hover and focus states for better keyboard navigation. Additionally, the use of aria-labels and role attributes where necessary provides further support for assistive technologies, promoting an inclusive and user-friendly experience.

Content Page

Technical Description

HTML Techniques

The page utilizes semantic HTML elements like <nav>, <section>, and proper heading tags (<h1>–<h3>) to divide long content into clear, manageable sections. An internal navigation menu (constructed with anchor links) allows users to jump directly to sections such as Introduction, Challenges, and Innovations, improving content accessibility. A visually striking banner with overlay text immediately captures attention, while the inclusion of a 'Go to Top' button (a fixed-position element) aids user navigation through extensive content.

CSS Techniques

The banner is styled with a background image and a semi-transparent overlay, ensuring that the overlaid text is legible while maintaining aesthetic appeal. Responsive design is implemented using flexible layouts (combining CSS Flexbox and Grid) to ensure that text and images adjust smoothly across devices. Additional CSS properties like scroll-behavior: smooth; enhance navigation, while the fixed 'Go to Top' button remains accessible at all times.

Purpose & Justification

This page is designed to present detailed content in an organized and engaging format. By leveraging semantic HTML and modern CSS layout techniques, the page remains user-friendly and accessible, even as it conveys extensive information on ocean conservation topics.

Accessibility

Accessibility is prioritized by using semantic HTML elements like <nav>, <section>, and proper heading hierarchy (<h1> to <h3>) to structure the content clearly for screen readers. The internal navigation menu, built with anchor links, allows users with assistive technologies to quickly jump to specific sections, enhancing usability. High-contrast text overlays on banners improve readability for visually impaired users, while the responsive design ensures accessibility across various devices. Additionally, the smooth scrolling feature (scroll-behavior: smooth;) provides a better experience for keyboard and screen reader users, and the fixed 'Go to Top' button enhances navigation by providing an easy way to return to the beginning of the page.

Challenges and Lessons Learned

Layout Consistency: Coordinating the design between different pages—especially integrating unique elements like animated loaders and interactive cards—required close collaboration and thorough testing. We resolved this by standardizing our approach using a shared Global CSS file and modular components.

Responsive Design and Cross-Browser Compatibility: Ensuring that the pages rendered well across different devices and browsers was initially challenging. We addressed this by leveraging CSS Flexbox, Grid layouts, and media queries, followed by rigorous testing on multiple platforms.

Interactive Element Debugging: Implementing features such as the meta refresh redirect and CSS animations required careful debugging. Through iterative testing and code review, we refined the animations and interactivity to function seamlessly.

Lessons Learned:
- The importance of planning and consistent coding practices cannot be overstated.
- Collaborative problem-solving and detailed code documentation help streamline the debugging process.
- Continuous testing and feedback lead to improvements in both design and functionality.
